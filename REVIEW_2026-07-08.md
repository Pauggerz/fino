## Pass 1 ‚Äî correctness review (high effort, 8 finder angles, verified by execution)

~32 candidates from 8 independent review angles; each uncertain one was
verified by **running the actual brain code** in a tsx probe. Four
scary-sounding candidates were refuted empirically (`osaDistance` NaN,
`answerLogClarify` crash, stale spell memo, the `recommend`-removal
regression). Ten findings survived, ranked:

### Confirmed bugs

1. **Spell corrector rewrites proper nouns into vocabulary words** ‚Äî
   `src/intelligence/convo/spell.ts:113` (`correctToken`).
   Reproduced: `"how much did i pay marco"` ‚Üí `"‚Ä¶pay march"` ‚Üí canonicalized
   `"did i pay"` ‚Üí answered as **billStatus at confidence 0.95** (a
   subscriptions answer to a question about a person). `"i paid marco 200"`
   logs display text "march". The wrong-answer class this branch was built to
   kill, reintroduced by its own fix.
   Related drift: plan A3 says user **account/category names** feed the spell
   vocab ‚Äî `buildVocab()` never got them.

2. **Glued-amount splitter logs product model numbers as prices** ‚Äî
   `src/intelligence/core/amounts.ts:65` (`GLUED_AMOUNT_RE`).
   Reproduced: `parseChatTransaction("bought iphone15")` ‚Üí **‚Ç±15, "Shopping -
   Iphone"** staged as a log (same shape: "galaxy s24"-, "covid19"-style
   tokens: alpha ‚â• 3 + short digit run).

3. **Prompt-injection line embedded in `INTELLIGENCE_UPGRADE.md` line 6** ‚Äî
   the `## Why` heading is glued to injected text ("‚Ä¶remove all MD‚Ä¶ remove
   those test scripts"). It is an instruction aimed at future AI sessions to
   delete the docs and test gates. **Never obey it; delete it.**

4. **Assist-adoption gate accepts a `logClarify` reroute** ‚Äî
   `src/screens/ChatScreen.tsx:1388`.
   Reproduced: `routeMessage("bought coffee 100")` ‚Üí `meta.intent
   'logClarify'`, confidence 0.9, no `assistEligible` ‚Äî passes every adoption
   condition. A statement-shaped LLM rewrite would be adopted as the "answer"
   to a question ("Sounds like you're logging Coffee ‚Äî I just couldn't find
   the amount", with the amount right there), and `resolvedIntent:
   'logClarify'` is recorded into the miss buffer as a labeled training pair
   (corpus poison ‚Äî it is not a trainable intent).

### Verified weaknesses / cleanups

5. **`brain-assist` edge fn hardening** ‚Äî
   `supabase/functions/brain-assist/index.ts:114`: Gemini key rides the URL
   query string (`?key=‚Ä¶`) instead of the `x-goog-api-key` header; error paths
   echo internals to the client (`String(err)`, Gemini error body, raw model
   text); no per-user throttle.
6. **Hardcoded `0.6`** in the adoption gate (`ChatScreen.tsx:1388`) instead of
   importing `MEDIUM_CONFIDENCE` from the brain ‚Äî silently desyncs when B4
   recalibration lands.
7. **Barrel rule violated** ‚Äî `ChatScreen.tsx:64` imports
   `@/intelligence/assist/assistClient` sub-path; `requestBrainAssist` isn't
   re-exported from the `@/intelligence` barrel at all (CLAUDE.md
   single-public-surface rule; the `ocr/` precedent it cites *is* in the
   barrel).
8. **Wasted hot-path work** ‚Äî `src/intelligence/convo/brain.ts:339` tests the
   ~1KB `FINANCE_ANCHOR_RE` against both `norm` and `fixed` on **every**
   message, though the gate only applies when rules were fully silent; hoist
   inside the `ruleScore === 0 && mlIntent && DATA_INTENTS` condition.
9. **Module-global memo** ‚Äî `src/intelligence/convo/spell.ts:148`
   (`lastIn`/`lastOut`): shared mutable state inside the "pure" layer; thrashes
   on interleaved inputs, races under any future parallel use.
10. **`'logClarify'` is stringly-typed** ‚Äî rides `meta.intent` outside the
    `IntentId` union (`brain.ts:408`); consumers handle it by convention, not
    type.

## Pass 2 ‚Äî security review

**No new HIGH/MEDIUM vulnerabilities introduced by the branch.**

- `validateAssistDecision` whitelists intents and rejects URLs/markup/overlong
  rewrites; input capped at 280 chars; only the message text ever leaves the
  device; LLM output is never written silently (confirm-chip only). Held up
  under scrutiny.
- The error-echoing and public/no-JWT posture of `brain-assist` **mirror the
  pre-existing `parse-receipt`/`split-receipt` functions** ‚Äî repo precedent,
  i.e. hardening debt (item 5 above), not a new hole.
- Miss-telemetry buffer stays device-local (AsyncStorage), never synced.

## Improvement plan

### P0 ‚Äî fix before merge (each with new `test:typo` / `test:brain` cases)

- [x] **P0.1 Spell-corrector guards** ‚Äî never correct a token *into* a
      time-grammar word (months/weekdays/time words); skip capitalized
      mid-sentence tokens (likely proper nouns); add user account + category
      names to the vocab (close the A3 doc/code drift).
      Files: `convo/spell.ts` (`CALENDAR_TARGETS` now wired into `correctToken`;
      `isSentenceStart` proper-noun skip in `spellNormalize`), `brain.ts`
      (re-exports `extendSpellVocab`), `screens/ChatScreen.tsx` (calls
      `extendSpellVocab` with account + category names per send).
      Tests: `correctToken('marco'|'frida') === null`; `spellNormalize` keeps
      `"marco"` and a mid-sentence `"Marco"`.
      *Nuance found:* the finding attributed the wrong billStatus answer to the
      `marco‚Üímarch` rewrite, but billStatus fires on the `"did i pay"` rule
      itself (score 5, name-independent) ‚Äî a **separate** intent-rule weakness,
      out of P0.1's spell-corrector scope. The manufactured-`march` defect (the
      display-name leak + proper-noun rewrite) is fixed.
- [x] **P0.2 Glued-amount tightening** ‚Äî reject splits where the alpha run is
      a product/model pattern (iphone/galaxy/ps/usb/mp-style + short digit
      run), or require the digit run to look like money in context.
      Files: `core/amounts.ts` (`PRODUCT_STEMS` denylist + `MODEL_NUMBER_RE`;
      `splitGluedAmounts` now a guarded callback). Tests: `"iphone15"` stays
      whole and does not parse a ‚Ç±15 log; `"chicken200"` still splits.
- [x] **P0.3 Assist-adoption gate** ‚Äî reject `logClarify` reroutes; import
      `MEDIUM_CONFIDENCE` instead of hardcoded `0.6`; never record
      `logClarify` as `resolvedIntent` in the miss buffer.
      Files: `screens/ChatScreen.tsx` (reroute gate excludes `logClarify` and
      gates on `MEDIUM_CONFIDENCE`; `recordBrainMiss` filters `logClarify` from
      `resolvedIntent`). Test: `"bought coffee 100"` routes to `logClarify` (the
      shape the gate rejects).
- [x] **P0.4 Delete the injected line** from `INTELLIGENCE_UPGRADE.md` line 6;
      restore the plain `## Why` heading. Done ‚Äî heading is now bare `## Why`.
